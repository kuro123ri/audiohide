AudioHide

LSB audio steganography — hide a song inside an image file. Runs entirely in your browser, no upload.

Settings

Image

Applies a JPEG pass before hiding data. Reduces pixel noise so LSB changes are less visually detectable. Output is always PNG.
1 = most compressed (blocky), 100 = no compression. Recommended: 60-85.
2-bit mode stores 2 bits in each colour channel instead of 1, doubling how much audio you can hide. Very hard to see but slightly more detectable by analysis tools.

Audio

Lower rate = smaller file = fits at lower speed = less pitch change. Trade-off is audio quality.
Boosts audio to maximum volume before encoding. Good for quiet source files.

Playback speed

Higher speed = smaller file but higher pitch. 1.0x = original pitch. Each doubling = +12 semitones (one octave up).

Decode

When extracting audio that was sped up to fit, automatically slow it back down to the original speed and pitch using the Web Audio API.

Scatter passkey (steganography protection)

Without a passkey, bits are written sequentially from pixel 0 — any LSB analysis tool detects the data instantly. With a passkey, bits are scattered across the image in a key-derived random order. Without the correct key the image looks like unmodified pixel noise.
The key is never stored or sent anywhere. You must remember it to decode.
How to use:
  1. Encode: Drop a PNG image and a song file, then click "Encode and Download".
  2. Decode: Drop an encoded PNG here or on the Decode tab, then click "Extract Audio".
  3. If the image is too small, the tool speeds up the audio to fit. Decode undoes this automatically.
  4. Always keep the output as PNG — saving as JPEG destroys the hidden data.
