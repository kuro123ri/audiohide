AudioHide

Hide a song inside an image file — runs entirely in your browser, no upload.

Settings

Image

Applies a JPEG pass before hiding data. Reduces pixel noise so LSB changes are less visually detectable. Output is always PNG.
Recommended: 60–85. Lower = smaller but blockier.
2-bit mode doubles capacity by storing 2 bits per colour channel. Very hard to see but slightly more detectable by analysis tools. Both sides must use the same setting.

Audio

Lower rate = smaller file = fits at lower speed = less pitch change. Trade-off is audio quality.
Boosts quiet audio to full volume. No quality loss.

Playback speed

1.0x = original pitch. Each doubling = +12 semitones (one octave up).

Decode

When audio was sped up to fit, automatically slows it back to the original speed and pitch on extraction.

Scatter passkey

Without a passkey, bits are written from pixel 0 in order — easy for steganalysis tools to detect. With a passkey the bits are scattered in a key-derived random order, making detection much harder.
Key is never stored or sent anywhere. You must remember it to decode.
How to use:
  1. Encode: Select a PNG image and a song file, then tap "Encode and Download".
  2. Decode: Drop or select an encoded PNG — it auto-detects and switches to the Decode tab.
  3. If the image is too small, audio is sped up automatically to fit. Decode reverses this.
  4. Always keep the output as PNG — re-saving as JPEG destroys the hidden data.
